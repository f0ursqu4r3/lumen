# Product

## Register

product

## Users

Lumen is for a solo developer using a self-hosted GitLab instance as a daily backlog and triage workspace. The use context is focused, repetitive issue work in an IDE-adjacent desktop environment, often during long dark-mode sessions.

## Product Purpose

Lumen is a fast, keyboard-friendly native shell over GitLab issues, merge requests, pipelines, labels, assignees, and saved views. It exists to make day-to-day GitLab work faster and calmer than the web UI while keeping GitLab as the source of truth and keeping tokens in the local host process.

## Brand Personality

Precise, calm, operational. The product should feel like trustworthy mission-control software: dense, scannable, fast, and restrained.

## Anti-references

Avoid marketing-site polish, decorative dashboards, heavy glass effects, gratuitous motion, and full-saturation color used as decoration. Avoid GitLab web UI heaviness, generic SaaS card grids, and UI that feels playful at the expense of triage speed.

## Design Principles

1. Make dense work scannable: preserve tight rows, predictable alignment, tabular metadata, and clear grouping.
2. Let color carry state: use hue for labels, status, priority, and the single primary signal, not decoration.
3. Keep GitLab close but calmer: expose the real workflow fields while reducing visual noise and navigation friction.
4. Respect the desktop shell: fast keyboard paths, native windows where useful, and resilient local behavior matter more than web-page ceremony.
5. Favor quiet confidence: interactions should feel precise and responsive without calling attention to themselves.

## Accessibility & Inclusion

Target practical WCAG AA contrast for text and controls, preserve keyboard access for issue triage, provide visible focus states, and keep motion short, state-driven, and safe under reduced-motion preferences.
